Práctica 1: Introducción a HTML 5

Diseño de una página web básica con HTML y CSS

Ejercicio 1: Análisis de una página web

En este ejercicio se pide que analices el código fuente de una página web sencilla. En concreto, la página web de la asignatura.

Apartado 1: Análisis del aspecto de la página

En primer lugar, revisa el aspecto general de la página:

  • Analiza la estructura y contenido de la página:
    • Qué información contiene y cómo está estructurada.
    • Piensa qué etiquetas utilizarías.
  • Analiza el formato y presentación:
    • ¿Qué cambios de formato identificas?
    • ¿Cómo los implementarías?
    • En concreto, fíjate en dos elementos y piensa cómo los implementarías:
      • La marca ¡NUEVO! que resalta las noticias.
      • La tabla de planificación, que marca en distintos colores las sesiones dependiendo del tipo de clases.

Apartado 2: Análisis de la implementación

Una vez familiarizado con el diseño de la página, visualiza el código fuente para analizar la implementación.

  • Sin contar las imágenes, ¿cuántos ficheros se utilizan para definir la página?
  • En el fichero index.html, revisa las etiquetas HTML utilizadas. Compáralas con las que habías seleccionado tú y justifica qué solución es más apropiada en cada caso.
  • ¿Qué otros ficheros se utilizan para la implementación? Indica su nombre y extensión.
  • Abre uno de los ficheros CSS enlazados desde index.html. ¿Por qué la sintaxis es distinta?
  • ¿Por qué crees que se utilizan dos ficheros CSS en vez de agrupar todas las reglas en uno?
  • Analiza cómo se especifica el formato de un elemento concreto. Por ejemplo, ¿cómo se hace para que las tablas tengan distintos formatos?

Apartado 3: Mejoras HTML 5

HTML 5 introduce nuevas etiquetas para definir la estructura y contenido de una página web. En este apartado veremos cómo se aplicarían en la página con la que estamos trabajando.

  • La información de la página se estructura en varios bloques: objetivos, programa, planificación, evaluación, etc. ¿Qué etiqueta HTML 5 usarías para definir cada uno de ellos?
  • Una de las ventajas de la etiqueta <NAV> es que mejora la accesibilidad. ¿En qué parte (o partes) de la página la utilizarías?
  • ¿En qué parte (o partes) de la página usarías la etiqueta <ARTICLE>? ¿Por qué?

Cuestionario entregable

Cuestionario en Aula Global

Se recomienda que cada componente de la pareja abra sesión en AG en un navegador distinto para que podáis rellenar ambos el cuestionario.

Ejercicio 2: Diseño y desarrollo de una página web

A lo largo del curso, se pide que desarrolléis dos proyectos en equipo. El objetivo de este ejercicio es diseñar la página web de dicho equipo, como si de una empresa se tratara.

Diseño

En primer lugar, pensad (¡sin escribir aún código!) qué información queréis publicar: nombre de la empresa, logo, información de contacto, personal, objetivos, áreas de trabajo, productos, etc. Dibujad en papel un diseño esquemático.

Contenidos

A continuación, plasmad dicha información de forma estructurada en un fichero HTML.

Crea un fichero de texto plano, con extensión html. Puedes usar cualquier editor de texto plano o bien Dreamweaver, que es un editor específico de páginas web. Si usas Dreamweaver, asegúrate de utilizar la pestaña de edición de código fuente.

Estructura la información utilizando las etiquetas adecuadas (no te preocupes por ahora por el formato):

  • Utiliza una etiqueta header para agrupar la información genérica de la empresa.
  • Utiliza un elemento nav para definir el menú de navegación de la página, con enlaces a los distintos subapartados.
  • Utiliza elementos article para encapsular contenidos susceptibles de ser sindicados por RSS (por ejemplo, ofertas de trabajo, alertas de nuevos productos, noticias sobre la empresa...)
  • Utiliza elementos section para definir los distintos bloques de información.
  • Utiliza encabezados de distintos niveles para definir los títulos.
  • Asegúrate de incluir enlaces (tanto a otros puntos de la página como a páginas externas) e imágenes.
  • Utiliza una etiqueta footer para definir el pie de página.
  • Añade metadatos a la página: título, sistema de codificación (utf-8), autor, etc.

Visualiza la página web creada. Observa que se aplica el formato por defecto.

Presentación (formato)

Recuerda que un buen diseño web separa el contenido de la presentación. Por tanto, en HTML 5 se recomienda especificar el formato mediante reglas CSS, prescindiendo de etiquetas de formato correspondientes a versiones anteriores del lenguaje.

Las reglas CSS pueden definirse incrustadas en la propia página HTML, mediante la etiqueta STYLE en la cabecera de la página (head). Escribe las reglas CSS necesarias para:

  • Definir el color de fondo de la página.
  • Cambiar el tipo de letra.
  • Cambiar el color de los títulos (encabezados)

La principal desventaja de definir las reglas CSS incrustadas en la propia página HTML, mediante la etiqueta STYLE, es que habría que repetirlas para cada una de las páginas de un sitio web si se quiere mantener un estilo uniforme. Por ello, lo más habitual es definirlas en un fichero aparte.

Crea un nuevo fichero, también de texto plano, pero con extensión CSS. Puedes editarlo con cualquier editor de texto plano, incluido Dreamweaver. Copia las reglas CSS que habías definido al nuevo fichero y elimínalas del fichero html. Enlaza el fichero CSS desde el fichero HTML, con la etiqueta LINK (en la cabecera head). Comprueba que el resultado es el mismo que cuando estaban definidas en la propia página.

Añade una regla para eliminar el subrayado de los enlaces. A cambio, define (mediante reglas css) un formato que los resalte en vez del subrayado de modo que el usuario pueda localizarlos fácilmente.

Añade libremente las reglas necesarias para formatear la página de acuerdo a tu diseño original.

Ejercicio Complementario: Formularios

El objetivo de esta ejercicio es diseñar un formulario web que pueda servir a modo de formulario de registro de usuarios de la empresa anterior. El formulario debe tener las siguientes características:

  • Estructuración del contenido usando nuevos elementos HTML5. Usará, al menos, los elementos header, section, footer
  • Uso de campos de formulario nuevo en HTML5. Al menos se usarán campos de tipo email, tel, date, range, number
  • Se comprobará el contenido del campo de teléfono mediante nuevos atributos de HTML5 y el uso de expresiones regulares (no mediante scripts JavaScript).
  • Se hará uso de hojas de estilo CSS para
    • poner borde rojo a los campos no válidos
    • poner boder verde al campo de email cuando su contenido sea válido
    • usar el efecto espejo sobre algún elemento
  • Se programará un script en JavaScript para que al enviar el formulario se muestre una alerta (alert) agradeciendo al usuario su participación. Por ejemplo, "Gracias Señor XX por su participación", siendo XX el nombre que el usuario introdujo en un campo del formulario.

Recomendaciones

  • Antes de comenzar a programar, dedicad 5 minutos a definir (¿en papel?) la estructura del formulario y los campos que vais a necesitar.
  • Existen varios editores de HTML en las aulas. Aunque puede utilizarse cualquier editor de texto plano, se recomienda el uso de Dreamweaver.
  • Durante la fase de implementación, se recomienda el uso de las herramientas para desarrolladores de Google Chrome, accesibles desde el menú Herramientas.

Preguntas de Autoevaluación

Tras la finalización de la práctica, deberíais ser capaces de contestar a estas preguntas:

  • Describe las ventajas de los siguientes artefactos de HTML5 que no estaban en versiones anteriores:
    • Campos de tipo tel, email, number, search, date, month, week, time
    • Los atributos del elemento input: required, autofocus
  • Enumera las diferencias y el uso específico de los siguientes artefactos HTML5:
    • Atributos placeholder y value del elemento input
    • Campos de tipo radio y checkbutton
    • Elementos meter y progress
  • ¿Para qué sirve el atributo pattern del elemento input? ¿Qué es una expresión regular? ¿Puede citar las ventajas de utilizar este atributo respecto a hacer la validación en el lado del servidor?
  • Al pulsar el botón de enviar, ¿a qué URL se accede? ¿se envían los datos en el formato adecuado?
